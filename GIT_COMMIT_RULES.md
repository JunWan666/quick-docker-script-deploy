# 提交规则

## 提交格式

统一使用：

```text
编号:类型:说明
```

示例：

```text
005:更新:优化 one-click 部署脚本服务选择
006:修复:修正 HTTP 文件服务启动提示
007:文档:更新 README 当前目录说明
```

## 类型

- `新增`：新增脚本、功能或配置
- `修复`：修复错误、兼容性问题或异常行为
- `更新`：调整已有功能、交互、默认值或输出说明
- `重构`：不改变功能前提下整理代码结构
- `文档`：更新 README、说明文档或注释
- `清理`：删除无用文件、旧目录或过期内容
- `回滚`：撤销某次改动

## 约定

- 每次提交尽量只做一类事情。
- 说明使用中文，尽量短，直接写清楚改了什么。
- 涉及部署行为变化时，同步更新 `README.md`。
- 不提交运行时生成目录，例如 `generated-stack/`、缓存目录和临时文件。
